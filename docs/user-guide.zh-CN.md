# Tabby 普通用户使用指南

> 面向日常使用场景，覆盖安装、首次配置与常见连接类型（SSH/串口/Telnet）。

## 0. 适用范围

- 平台：Windows 10、macOS、Linux
- 场景：本地终端、远程 SSH、串口设备与 Telnet/Socket 连接

## 1. 安装与更新

### 1.1 Windows / macOS

1. 进入项目 Releases 页面下载对应平台安装包。
2. 运行安装包并完成安装向导。
3. 启动 Tabby。

### 1.2 Linux

1. 根据官方仓库（Debian/Ubuntu 或 RPM）说明添加软件源。
2. 安装 Tabby 包并启动。
3. 如需更新，按系统包管理器的更新流程操作。

## 2. 首次启动与基础设置

**界面预览**

![Tabby 主界面](readme.png)

### 2.1 选择默认终端（本地）

1. 打开 Tabby 设置页面。
2. 在“Profiles/终端配置”区域新建或选择本地 Shell 配置。
3. 选择目标 Shell（如 PowerShell、WSL、Git-Bash、CMD 等）。
4. 保存并设置为默认配置。

### 2.2 字体与外观

1. 打开“外观/Appearance”设置。
2. 设置字体与字号，必要时启用/关闭字体连字。
3. 选择主题或配色方案。

### 2.3 快捷键

1. 打开“快捷键/Hotkeys”设置。
2. 搜索要修改的动作并重新绑定。
3. 需要多键序列时，使用多键快捷键（multi-chord）。

## 3. 基本操作

### 3.1 标签与分屏

1. 新建终端标签页。
2. 使用分屏功能在同一窗口打开多个终端。
3. 关闭不需要的标签或分屏。

![终端与分屏示意](readme-terminal.png)

### 3.2 复制与粘贴

1. 在终端中选择文本并复制。
2. 在设置中可启用“右键粘贴/选中即复制”等习惯行为。

### 3.3 会话恢复

1. Tabby 会记住你的标签与布局。
2. 重新启动后可继续之前的会话。

## 4. SSH 连接

![SSH 连接示意](readme-ssh.png)

### 4.1 新建 SSH 连接

1. 打开设置中的“连接/Profiles”管理区。
2. 新建 SSH 配置。
3. 填写 Host、Port、Username。
4. 选择认证方式：密码或 SSH 密钥（可选使用系统代理）。
5. 测试连接并保存。

### 4.2 常用高级功能（可选）

1. 跳板机（Jump Host）。
2. 端口转发与 X11 转发。
3. Agent 转发与登录脚本。

## 5. 串口连接

1. 新建串口配置。
2. 选择串口设备与波特率。
3. 配置数据位、校验位、停止位等参数。
4. 连接后根据需要启用换行转换或十六进制输入。

## 6. Telnet / Socket 连接

1. 新建 Telnet/Socket 配置。
2. 填写 Host 与 Port。
3. 保存并连接。

## 7. 文件传输（Zmodem）

1. 确认目标主机支持 Zmodem（常见为 lrzsz 工具）。
2. 在 SSH 会话中执行 `sz` 或 `rz`。
3. Tabby 会触发文件传输流程。

## 8. 插件与主题

### 8.1 安装插件

1. 打开设置中的“Plugins”。
2. 搜索并安装所需插件。
3. 如需，重启 Tabby 使插件生效。

### 8.2 主题与配色

1. 安装主题/配色插件（如社区配色）。
2. 在“外观/Appearance”中选择应用。

## 9. 便携模式（Windows）

1. 找到 `Tabby.exe` 所在目录。
2. 在同一目录创建 `data` 文件夹。
3. 启动 Tabby，配置将写入 `data`。

## 10. Web 版本

1. 访问 `https://tabby.sh/app` 使用 Web 版本。
2. 如需自托管，可参考 tabby-web 项目。

## 11. 常见问题

1. SSH 连接失败：检查网络、端口、用户名与密钥格式。
2. 字体显示异常：切换字体或关闭连字。
3. 快捷键冲突：在 Hotkeys 中调整。
4. 插件不生效：确认安装完成后重启应用。
5. Linux 启动异常：检查系统依赖库是否完整。
